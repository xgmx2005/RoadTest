# 道路信息检测期末实验报告

## 1. 实验目标
本实验面向司机视角道路图像，完成以下任务：

1. 检测图像中的车道线；
2. 判断每条车道线的颜色，只区分为**黄色**和**白色**两类；
3. 对每张图输出可视化结果，在图中标出车道线及其颜色；
4. 支持批量处理 50~100 张图片，并统计平均单张图像的处理时间；
5. 对测试结果进行人工统计，计算车道线检测的 precision、recall，以及黄色车道线分类的 precision、recall。

---

## 2. 方法设计

### 2.1 总体思路
本实验采用“**预训练车道线检测模型 + 颜色后处理分类**”的方案。

- 车道线检测部分：采用基于 **CULane** 数据集训练的 **Ultra-Fast-Lane-Detection（UFLD）** 预训练模型；
- 颜色分类部分：在检测出的车道线点附近采样像素，结合亮像素筛选、BGR 通道比值与 HSV 颜色空间信息，将车道线分类为黄色或白色；
- 可视化部分：在原图上绘制每条车道线，并在对应位置标注 `yellow` 或 `white`；
- 批量处理部分：对指定文件夹内图像进行循环推理，输出结果图与 `results.json` 统计文件，同时计算总耗时与平均耗时。

该方案的优点是：
1. 可直接复用公开数据集上的成熟模型，满足课程允许使用已有权重的要求；
2. 推理速度快，适合批量测试；
3. 实现成本较低，便于完成可视化与统计分析。

---

## 3. 模型与实现

### 3.1 车道线检测模型
本实验使用 **UFLD（Ultra-Fast-Lane-Detection）** 作为车道线检测主模型，并使用 **CULane ResNet18** 预训练权重。

模型推理流程如下：
1. 将输入图像缩放到模型所需尺寸；
2. 输入网络得到车道线栅格输出；
3. 对输出进行 softmax 和位置解码；
4. 恢复到原图尺寸后得到每条车道线的点集。

### 3.2 颜色分类方法
由于公开预训练模型主要解决“车道线位置检测”，并不直接输出车道线颜色，因此本实验对每条检测出的车道线做后处理分类：

1. 在每条车道线的若干采样点附近截取局部区域；
2. 对局部区域进行亮像素筛选，尽量去除深色路面背景；
3. 统计像素颜色特征：
   - 使用 BGR 通道比值区分黄色与白色；
   - 结合 HSV 中的 hue（色调）和 saturation（饱和度）进一步判断；
4. 当蓝色分量相对较低、饱和度较高且色调位于黄色区间时，判为黄色；否则判为白色。

这种方法实现简单、速度快，但在强逆光、暖色光照、路面反光等情况下容易受到干扰。

### 3.3 可视化输出
对每张图像，程序都会生成一张带标注的结果图，内容包括：
- 车道线点和折线；
- 每条车道线的颜色标签；
- 当前图像检测到的车道线总数、黄线数、白线数。

### 3.4 批量处理与统计
程序支持递归读取文件夹中的图像，并对每张图执行：
- 车道线检测；
- 黄/白分类；
- 可视化输出；
- 统计推理时间。

最终程序将结果写入 `results.json`，其中包括：
- 每张图片的车道线数量；
- 黄线与白线数量；
- 每张图片的处理时间；
- 所有图片的总处理时间、平均处理时间和 FPS。

## 7. 人工评估与指标计算
根据题目要求，需要给出：
1. 车道线检测的 precision 和 recall；
2. 黄色车道线分类的 precision 和 recall。

本实验按照“**只关注司机当前行驶方向主车道**，忽略远处非主路、严重遮挡区域和不确定边界”的原则，对 `test-difficult` 数据集进行了人工保守统计。

### 7.1 车道线检测指标
人工统计结果为：
- 有效预测车道线数：89
- 其中正确检测（TP）：83
- 错误检测（FP）：6
- 漏检（FN）：2

由此可得：

\[
Precision = \frac{TP}{TP + FP} = \frac{83}{83 + 6} = 0.933
\]

\[
Recall = \frac{TP}{TP + FN} = \frac{83}{83 + 2} = 0.976
\]

即：
- **车道线检测精确率：93.3%**
- **车道线检测召回率：97.6%**

### 7.2 黄色车道线分类指标
对清晰可见的黄色车道线做保守人工统计，结果为：
- 黄色线 TP：6
- 黄色线 FP：3
- 黄色线 FN：4

由此可得：

\[
Yellow\ Precision = \frac{6}{6 + 3} = 0.667
\]

\[
Yellow\ Recall = \frac{6}{6 + 4} = 0.600
\]

即：
- **黄色线精确率：66.7%**
- **黄色线召回率：60.0%**

## 8. 结果分析

### 8.1 车道线几何检测效果较好
从实验结果可以看出，车道线检测本身的 precision 和 recall 都较高。这说明基于 CULane 预训练的 UFLD 模型已经能够较稳定地提取司机视角道路中的主要车道边界，尤其对高速、城市主干道等场景有较好的适应能力。

### 8.2 颜色分类难度高于几何检测
黄色车道线分类的指标明显低于车道线几何检测指标，主要原因有：
1. 颜色分类不是由深度模型直接输出，而是通过后处理像素统计实现；
2. 黄色和白色在逆光、黄昏、暖色偏移等条件下分布容易重叠；
3. 车头反光、路面老化、阴影和污染会干扰采样像素；
4. 有些白线在暖光条件下会呈现偏黄色，导致误判。

### 8.3 运行效率满足实验要求
题目要求考虑运行效率，并能够一次性处理 50~100 张图像。从结果看：
- 87 张图像可以稳定批量运行；
- 平均单张图像耗时约 20~30 ms；
- 在 CPU 环境下也能达到较高处理速度。

因此，该方案满足课程实验对效率的要求。

---

## 9. 方法优点与不足

### 9.1 优点
1. 直接使用 CULane 预训练模型，工程实现简单；
2. 无需重新训练即可完成车道线检测任务；
3. 支持可视化输出和批量统计；
4. 推理效率较高，便于实验展示和答辩。

### 9.2 不足
1. 颜色分类依赖后处理，精度受光照影响较大；
2. 个别场景中护栏边缘、文字区域、反光区域可能造成误检；
3. 对复杂遮挡和极端光照场景仍存在漏检或错分情况；
4. 目前未使用人工标注文件进行严格自动评测，仍以人工保守判断为主。

---

## 10. 可改进方向
未来可从以下几个方面改进：

1. **颜色归一化**：在分类前进行白平衡或颜色校正，提高黄/白区分能力；
2. **更精细的 ROI 限制**：剔除车头区域、护栏区域和路面文字区域，减少误检；
3. **端到端联合学习**：将颜色分类与车道线检测统一到同一网络中，提高整体鲁棒性；
4. **人工标注评测集**：建立更规范的 GT 标注，提高 precision / recall 统计的客观性；
5. **透视变换**：结合鸟瞰变换改善车道线几何结构分析。

---

## 11. 结论
本实验基于 CULane 预训练的 UFLD 模型，完成了司机视角道路图像中的车道线检测与黄/白颜色分类任务，实现了：
- 车道线检测；
- 车道线颜色分类；
- 可视化输出；
- 批量处理与效率统计；
- 人工评估 precision / recall。

实验结果表明：
- 车道线几何检测效果较稳定；
- 黄/白分类能够基本完成任务，但在复杂光照条件下仍有改进空间；
- 整体方法具有较好的工程可实现性和较高的运行效率，满足本次期末实验的基本要求。
