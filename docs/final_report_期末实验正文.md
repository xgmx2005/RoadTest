# 道路信息检测期末实验报告

## 1. 实验目标
本实验面向司机视角道路图像，完成以下任务：

1. 检测图像中的车道线；
2. 判断每条车道线的颜色，只区分为**黄色**和**白色**两类；
3. 对每张图输出可视化结果，在图中标出车道线及其颜色；
4. 支持批量处理 50~100 张图片，并统计平均单张图像的处理时间；
5. 对测试结果进行人工统计，计算车道线检测的 precision、recall，以及黄色车道线分类的 precision、recall。

---

## 2. 方法设计

### 2.1 总体思路
本实验采用“**预训练车道线检测模型 + 颜色后处理分类**”的方案。

- 车道线检测部分：采用基于 **CULane** 数据集训练的 **Ultra-Fast-Lane-Detection（UFLD）** 预训练模型；
- 颜色分类部分：在检测出的车道线点附近采样像素，结合亮像素筛选、BGR 通道比值与 HSV 颜色空间信息，将车道线分类为黄色或白色；
- 可视化部分：在原图上绘制每条车道线，并在对应位置标注 `yellow` 或 `white`；
- 批量处理部分：对指定文件夹内图像进行循环推理，输出结果图与 `results.json` 统计文件，同时计算总耗时与平均耗时。

该方案的优点是：
1. 可直接复用公开数据集上的成熟模型，满足课程允许使用已有权重的要求；
2. 推理速度快，适合批量测试；
3. 实现成本较低，便于完成可视化与统计分析。

---

## 3. 模型与实现

### 3.1 车道线检测模型
本实验使用 **UFLD（Ultra-Fast-Lane-Detection）** 作为车道线检测主模型，并使用 **CULane ResNet18** 预训练权重。

模型推理流程如下：
1. 将输入图像缩放到模型所需尺寸；
2. 输入网络得到车道线栅格输出；
3. 对输出进行 softmax 和位置解码；
4. 恢复到原图尺寸后得到每条车道线的点集。

### 3.2 颜色分类方法
由于公开预训练模型主要解决“车道线位置检测”，并不直接输出车道线颜色，因此本实验对每条检测出的车道线做后处理分类：

1. 在每条车道线的若干采样点附近截取局部区域；
2. 对局部区域进行亮像素筛选，尽量去除深色路面背景；
3. 统计像素颜色特征：
   - 使用 BGR 通道比值区分黄色与白色；
   - 结合 HSV 中的 hue（色调）和 saturation（饱和度）进一步判断；
4. 当蓝色分量相对较低、饱和度较高且色调位于黄色区间时，判为黄色；否则判为白色。

这种方法实现简单、速度快，但在强逆光、暖色光照、路面反光等情况下容易受到干扰。

### 3.3 可视化输出
对每张图像，程序都会生成一张带标注的结果图，内容包括：
- 车道线点和折线；
- 每条车道线的颜色标签；
- 当前图像检测到的车道线总数、黄线数、白线数。

### 3.4 批量处理与统计
程序支持递归读取文件夹中的图像，并对每张图执行：
- 车道线检测；
- 黄/白分类；
- 可视化输出；
- 统计推理时间。

最终程序将结果写入 `results.json`，其中包括：
- 每张图片的车道线数量；
- 黄线与白线数量；
- 每张图片的处理时间；
- 所有图片的总处理时间、平均处理时间和 FPS。

## 7. 人工评估与指标计算
根据题目要求，需要给出：
1. 车道线检测的 precision 和 recall；
2. 黄色车道线分类的 precision 和 recall。

为客观评估，本节以**老师提供的标注真值**（每张图的车道线总数、白线数、黄线数）为基准，对 `test-difficult`（37 张）进行统计。`test(1).zip` 中重命名的 `1.jpg~37.jpg` 与本数据集图片按排序一一对应（像素比对 MSE=0，已验证）。

判定标准：模型检出的一条线，需同时满足 (1) 压在真实车道线上、(2) 方向偏差不超过约 15°、(3) 颜色判断正确，才计为“正确（TP）”。
- Precision = 正确数 TP / 模型检测数；
- Recall = 正确数 TP / 老师真值数。

### 7.1 总体指标（对照老师真值）

| 项目 | 老师真值 | 模型检测 | 正确(TP) | Precision | Recall |
|------|------|------|------|------|------|
| 白线 | 106 | 77 | 57 | 57/77 = **74.0%** | 57/106 = **53.8%** |
| 黄线 | 12 | 14 | 6 | 6/14 = **42.9%** | 6/12 = **50.0%** |
| 全部车道线 | 118 | 91 | 63 | 63/91 = **69.2%** | 63/118 = **53.4%** |

### 7.2 白线 / 黄线 各自指标

白色车道线：
\[
White\ Precision = \frac{57}{77} = 0.740,\qquad White\ Recall = \frac{57}{106} = 0.538
\]

黄色车道线：
\[
Yellow\ Precision = \frac{6}{14} = 0.429,\qquad Yellow\ Recall = \frac{6}{12} = 0.500
\]

即：
- **白线精确率 74.0%，召回率 53.8%**
- **黄线精确率 42.9%，召回率 50.0%**

逐图 37 行明细（真白/真黄、检白/检黄、对白/对黄、备注）见 `docs/对照老师真值_统计对比.md`。

## 8. 结果分析

### 8.1 召回率偏低：主要源于漏检外侧车道
对照老师真值后，整体召回率约 53%。主要原因是：老师按**整幅路面**标注，多车道高速/快速路一张图常有 4~5 条车道线（如图 3/17/18/25/26 真值为 5 条），而 UFLD(CULane) 默认聚焦**本车道附近**，通常只稳定输出 2~3 条，外侧相邻车道与远处车道容易漏检。这是召回率低于精确率的主要原因。

### 8.2 精确率扣分：乡道路缘误检与颜色判错
精确率约 69% 的失分集中在两类：
1. **乡道把"路缘"当车道线**：图 10/11/12/16 真值白线=0（仅一条黄色中心线），但模型把柏油路与土路/路肩的边界误当成白色车道线，产生多余误检；
2. **黄/白颜色判错**：黄色中心线/黄实线在部分图中被判成白（图 11/13/14/22/34/37），少数白线在暖光下被判成黄（图 18），护栏、路缘、强逆光区还会出现零碎的黄色误检。

### 8.3 颜色分类难度高于几何检测
黄线指标（P 42.9% / R 50.0%）明显低于白线，主要原因有：
1. 颜色分类不是由深度模型直接输出，而是通过后处理像素统计实现；
2. 黄色和白色在逆光、黄昏、暖色偏移等条件下分布容易重叠；
3. 车头反光、路面老化、阴影和污染会干扰采样像素；
4. 有些白线在暖光条件下会呈现偏黄色，导致误判。

### 8.4 运行效率满足实验要求
题目要求考虑运行效率，并能够一次性处理 50~100 张图像。从结果看：
- 87 张图像可以稳定批量运行；
- 平均单张图像耗时约 20~30 ms；
- 在 CPU 环境下也能达到较高处理速度。

因此，该方案满足课程实验对效率的要求。

## 9. 方法优点与不足

### 9.1 优点
1. 直接使用 CULane 预训练模型，工程实现简单；
2. 无需重新训练即可完成车道线检测任务；
3. 支持可视化输出和批量统计；
4. 推理效率较高，便于实验展示和答辩。

### 9.2 不足
1. 颜色分类依赖后处理，精度受光照影响较大；
2. 个别场景中护栏边缘、文字区域、反光区域可能造成误检；
3. 对复杂遮挡和极端光照场景仍存在漏检或错分情况；
4. 目前未使用人工标注文件进行严格自动评测，仍以人工保守判断为主。

---

## 10. 可改进方向
未来可从以下几个方面改进：

1. **颜色归一化**：在分类前进行白平衡或颜色校正，提高黄/白区分能力；
2. **更精细的 ROI 限制**：剔除车头区域、护栏区域和路面文字区域，减少误检；
3. **端到端联合学习**：将颜色分类与车道线检测统一到同一网络中，提高整体鲁棒性；
4. **人工标注评测集**：建立更规范的 GT 标注，提高 precision / recall 统计的客观性；
5. **透视变换**：结合鸟瞰变换改善车道线几何结构分析。

---

## 11. 结论
本实验基于 CULane 预训练的 UFLD 模型，完成了司机视角道路图像中的车道线检测与黄/白颜色分类任务，实现了：
- 车道线检测；
- 车道线颜色分类；
- 可视化输出；
- 批量处理与效率统计；
- 人工评估 precision / recall。

对照老师标注真值，在 `test-difficult`（37 张）上的指标为：
- 白线：Precision 74.0%、Recall 53.8%；
- 黄线：Precision 42.9%、Recall 50.0%；
- 全部车道线：Precision 69.2%、Recall 53.4%。

实验结果表明：
- 本车道附近的主要车道线检测较稳定，召回率偏低主要因外侧/远处车道漏检；
- 黄/白分类能够基本完成任务，但在复杂光照、乡道路缘等条件下仍有改进空间；
- 整体方法具有较好的工程可实现性和较高的运行效率，满足本次期末实验的基本要求。
