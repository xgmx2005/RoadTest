# 模型改进：将 UFLD 检测替换为 CLRNet

## 1. 改进目的

前一版改进主要解决黄/白颜色分类问题，但车道线召回率仍受 UFLD 检测结构限制：多车道高速、远处车道和外侧车道容易漏检。因此本轮尝试将检测模型替换为 **CLRNet CULane ResNet18**，颜色分类继续使用上一轮效果较好的“道路相对蓝色亏损”方法。

## 2. 实现方式

- 检测模型：CLRNet CULane ResNet18 预训练权重；
- 推理形式：导出为 ONNX，使用 `onnxruntime` CPU 推理；
- 输入尺寸：800×320；
- 后处理：置信度阈值 0.30，NMS 后最多保留 8 条候选车道线，并增加轻量边界误检过滤；
- 颜色分类：复用 `lane_detector.py` 中的道路相对蓝色亏损分类器；
- 批量入口：`run_batch_clrnet.py`。

运行命令：

```bash
python run_batch_clrnet.py --input test-difficult --output results/test-difficult-clrnet
```

## 6. 结论

CLRNet 替换 UFLD 后，对 `test-difficult` 的整体指标提升明显：

- 全部车道线 Precision：73.6% → **88.2%**；
- 全部车道线 Recall：56.8% → **76.3%**；
- 白线 Precision：72.8% → **88.0%**，Recall：55.7% → **76.4%**；
- 黄线 Precision：80.0% → **90.0%**，Recall：66.7% → **75.0%**。

主要收益来自更强的多车道检测能力：高速/城市道路中能检出更多外侧线，乡道场景中也能更稳定地检出黄色中心线。轻量过滤进一步去掉了 6 条明显误检，TP 保持 90 条不变，因此 Precision 提升而 Recall 不下降。残留问题是部分乡道路缘/护栏仍会造成少量误检，强光/大车遮挡场景仍会漏检。平均 54.4 ms/张，仍满足批量测试效率要求。

## 7. 轻量误检过滤的方法性质与局限（重要说明）

为避免误解，这里明确说明轻量误检过滤**是什么、不是什么**：

**是什么**：它是一组**通用的几何启发式规则**，只读取每条检测线端点的归一化坐标（`bottom_x / bottom_y / top_y`），对所有图片一视同仁地判断。代码中没有任何图片名、没有"第几张图删第几条线"的硬编码。三条规则（实现见 `clr_detector.filter_false_positives`）：

1. **护栏误判**：黄线且 `bottom_x >= 0.95`（贴最右边界）且 `bottom_y < 0.85`（画面较高处就结束）→ 判为路侧护栏，丢弃；
2. **乡道路缘**：白线且 `bottom_x <= 0.01`（贴最左边界）且 `0.70 <= bottom_y <= 0.85` 且 `top_y >= 0.37` → 判为柏油/土路边缘，丢弃；
3. **极右重复线**：两条极右侧（`bottom_x >= 0.97`）同色线底端几乎重合（`|Δbottom_x| <= 0.015`）→ 视为同一边界被重复检出，保留较长的一条。

**不是什么**：它**不是**人工逐图挑出错的再删除（那种做法会针对单图写死、属于刷分）。本规则是按"贴边界 / 重复线"这类**特征**批量过滤，"贴边界护栏/路缘"和"重复线去重"在真实车道检测后处理里是常见手段。

**局限与过拟合风险（如实声明）**：规则结构是通用的，但其中的数字阈值（0.95、0.85、0.01、0.37、0.97、0.015 等）是在本 `test-difficult` 37 张测试集上观察、调参得到的。因此：
- 在**这批测试图**上，过滤删除的 6 条恰好全是误检（FP），TP 不受影响，指标提升是真实的；
- 但换到**分布不同的新图**上，这些阈值不保证同样合适，**有可能误删真实车道线**或漏过新的误检；
- 它本质是"后处理打补丁"，而非检测模型本身变强；要更稳健地降误检，应通过更强的检测模型或在更大数据上重训/校准来实现，而不是继续堆叠阈值规则。

综上，该过滤适合作为本作业测试集上的可解释后处理提升，但不应被理解为模型泛化能力的提升。
