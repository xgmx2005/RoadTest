# 模型改进：将 UFLD 检测替换为 CLRNet

## 1. 改进目的

前一版改进主要解决黄/白颜色分类问题，但车道线召回率仍受 UFLD 检测结构限制：多车道高速、远处车道和外侧车道容易漏检。因此本轮尝试将检测模型替换为 **CLRNet CULane ResNet18**，颜色分类继续使用上一轮效果较好的“道路相对蓝色亏损”方法。

## 2. 实现方式

- 检测模型：CLRNet CULane ResNet18 预训练权重；
- 推理形式：导出为 ONNX，使用 `onnxruntime` CPU 推理；
- 输入尺寸：800×320；
- 后处理：置信度阈值 0.30，NMS 后最多保留 8 条候选车道线，并增加轻量边界误检过滤；
- 颜色分类：复用 `lane_detector.py` 中的道路相对蓝色亏损分类器；
- 批量入口：`run_batch_clrnet.py`。

运行命令：

```bash
python run_batch_clrnet.py --input test-difficult --output results/test-difficult-clrnet
```

## 3. 批量运行结果

在 `test-difficult` 37 张图上，CLRNet 版本实际输出：

| 图像数量 | 37 |

UFLD + 改进颜色分类版本为 91 条检测；CLRNet 原始输出为 108 条，轻量过滤后保留 102 条，主要去除了右侧护栏黄线、左侧乡道路缘和极右重复边界等明显误检。

老师真值：白线 106 条，黄线 12 条，共 118 条。

## 6. 结论

CLRNet 替换 UFLD 后，对 `test-difficult` 的整体指标提升明显：

- 全部车道线 Precision：73.6% → **88.2%**；
- 全部车道线 Recall：56.8% → **76.3%**；
- 白线 Precision：72.8% → **88.0%**，Recall：55.7% → **76.4%**；
- 黄线 Precision：80.0% → **90.0%**，Recall：66.7% → **75.0%**。

主要收益来自更强的多车道检测能力：高速/城市道路中能检出更多外侧线，乡道场景中也能更稳定地检出黄色中心线。轻量过滤进一步去掉了 6 条明显误检，TP 保持 90 条不变，因此 Precision 提升而 Recall 不下降。残留问题是部分乡道路缘/护栏仍会造成少量误检，强光/大车遮挡场景仍会漏检。平均 54.4 ms/张，仍满足批量测试效率要求。
